# Evaluation Set

Use these tasks to test whether the skill creates Human 2.0 images that are practical, branded, and visually strong.

## Eval 1: OpenClaw update

Input: article about an OpenClaw release with fixes for covers, Codex, Telegram media, and security.

Expected:

- 16:9 cover
- bold headline about the main update
- clean white panel with 3-4 update cards
- or one stronger product/update scene with cards as satellites
- H2.0 badge bottom-left
- no overloaded changelog table

## Eval 2: HyperFrames

Input: article about HyperFrames turning HTML scenes into MP4 via headless Chrome and FFmpeg.

Expected:

- headline like `HYPERFRAMES: ТВОЙ ИИ-МОНТАЖЁР`
- visual flow: HTML card -> tool node -> MP4 preview card
- no dense code text
- clean Human 2.0 palette

## Eval 3: Agent First community

Input: article arguing that communities should be agent-first, with memory, skills, transcripts, and agent access.

Expected:

- not a dense community diagram
- visual metaphor: human + agent in center, connected to memory/skills/search/text cards
- 16:9 or 9:16 depending on request
- no `AI` in Russian text

## Eval 4: Course vs environment

Input: article: `Курс по ИИ умер. Да здравствует среда внедрения`.

Expected:

- premium object cover allowed
- crown metaphor
- headline readable
- gold used intentionally
- Human 2.0 logo/badge integrated
- no generic online course imagery

## Eval 7: Claude-blog SEO/AI answers

Input: article about `claude-blog` turning article writing into a verifiable pipeline for Google and AI answers.

Expected:

- 16:9 Dzen cover
- headline like `БЛОГ ПОД GOOGLE И ИИ-ОТВЕТЫ`
- GPT Image 2 creates the main cover, not a code-drawn template
- visual metaphor: finished article/browser artifact being scanned, cited, or validated
- GitHub/README screenshot may be a small framed proof point, not the entire composition
- no flat `plan -> article -> schema -> review` card row as the main visual

## Eval 5: Data infographic

Input: chart/table about ИИ-industries vs others, salary premium, productivity, or adoption.

Expected:

- clean infographic mode
- 2-4 cards or one simple chart
- exact numbers only from input
- Human 2.0 colors
- no fake extra data

## Eval 6: Reels vertical conversion

Input: existing horizontal Agent First cover, request to make it vertical.

Expected:

- 9:16 vertical composition
- headline in top safe zone
- visual in middle
- H2.0 footer in bottom safe zone
- no horizontal text cut off

## Pass criteria

A generated image passes when:

- the main claim is clear in 2 seconds
- the image looks like Human 2.0, not generic SaaS
- it visibly matches one Reference DNA archetype
- text is short and readable
- no Russian spelling artifacts are obvious
- no critical text/logo is outside safe zones
- visual metaphor fits the article topic
- the cover has a strong scroll-stopping visual hook, not only a neat diagram
- the main composition was generated, not rebuilt in code
- there are no unnecessary extra labels, buttons, tables, or random logos
