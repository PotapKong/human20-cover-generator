# Headline Patterns

Use this reference before choosing the final image headline. The goal is not only to sound smart, but to make the cover understandable in a feed within 2 seconds.

## Priority ladder

For Dzen, Telegram, Instagram, and broad-audience visuals, choose headline logic in this order:

1. User benefit
2. Real-world use case
3. Tool or product name
4. Technical proof
5. Internal implementation details

Default rule: if a non-technical reader would not understand why the tool matters, the headline is too insider.

## Audience modes

### Mass audience

Use for Dzen, Instagram, Telegram, and posts for entrepreneurs / operators / creators.

Headline should answer:

- What can I do with this?
- What pain does it remove?
- Why should I care now?

Good:

- `БЫСТРАЯ РАСШИФРОВКА АУДИО`
- `ЛОКАЛЬНАЯ ТРАНСКРИБАЦИЯ АУДИО`
- `ДЛИННЫЕ АУДИО БОЛЬШЕ НЕ НУЖНО ЖДАТЬ`
- `ВЫШЕЛ БЫСТРЫЙ WHISPER ДЛЯ GPU И MAC`
- `КАК БЫСТРО РАСШИФРОВАТЬ ДЛИННОЕ АУДИО`

Avoid as main headline:

- `WHISPER НА УСКОРЕНИИ`
- `ТРАНСКРИБАЦИЯ НА MPS`
- `FLASH ATTENTION ДЛЯ WHISPER`
- `CLI ДЛЯ BATCHING`

### Editorial / expert audience

Use when the audience already follows ИИ tools and understands product names.

Good:

- `WHISPER ДЛЯ ТЕХ, КТО НЕ ЛЮБИТ ЖДАТЬ`
- `HYPERFRAMES: ВИДЕО СОБИРАЕТ АГЕНТ`
- `OPENCLAW БЛИЖЕ К ЖИВОМУ АССИСТЕНТУ`
- `КОНТРОЛЬ НАД СТЕКОМ ВАЖНЕЕ УДОБНОГО ВХОДА`

### Technical audience

Use only when the post is clearly for developers or the user asks for a technical cover.

Allowed:

- CLI
- MCP
- GPU
- Mac
- local model
- API
- Docker
- FFmpeg

Still avoid leading with implementation details if the benefit is more useful.

## Headline formulas

### A. Benefit first

Best for broad coverage and Dzen covers.

- `<ПОНЯТНОЕ ДЕЙСТВИЕ>`
- `<БОЛЬШЕ НЕ НУЖНО ЖДАТЬ>`
- `<РУТИНА СТАЛА БЫСТРЕЕ>`

Examples:

- `БЫСТРАЯ РАСШИФРОВКА АУДИО`
- `БЫСТРЫЕ ВИДЕО ИЗ ГОТОВОГО ТЕКСТА`
- `СОЗВОНЫ ТЕПЕРЬ МОЖНО РАЗОБРАТЬ БЫСТРЕЕ`

### B. Problem to solution

Best for clickability.

- `УСТАЛИ ЖДАТЬ <РЕЗУЛЬТАТ>?`
- `<БОЛЬ> ТЕПЕРЬ РЕШАЕТСЯ БЫСТРЕЕ`
- `<СТАРЫЙ ПРОЦЕСС> БОЛЬШЕ НЕ ТЯНЕТ ВРЕМЯ`

Examples:

- `УСТАЛИ ЖДАТЬ РАСШИФРОВКУ?`
- `ДЛИННЫЕ ЗАПИСИ БОЛЬШЕ НЕ ТОРМОЗЯТ РАБОТУ`
- `КУРСЫ ПО ИИ УСТАРЕВАЮТ БЫСТРЕЕ`

### C. Tool plus benefit

Best when the product name matters.

- `<TOOL> ДЛЯ <ПОНЯТНОЙ ЗАДАЧИ>`
- `<TOOL> ТЕПЕРЬ <ПРАКТИЧЕСКИЙ ЭФФЕКТ>`
- `ВЫШЕЛ <ПОНЯТНЫЙ TOOL> ДЛЯ <АУДИТОРИЯ / ПЛАТФОРМА>`

Examples:

- `БЫСТРЫЙ WHISPER ДЛЯ GPU И MAC`
- `WHISPER ДЛЯ БЫСТРОЙ РАСШИФРОВКИ`
- `INSANELY FAST WHISPER: БЫСТРАЯ РАСШИФРОВКА`

### D. Strong claim

Best for opinion essays and concept posts.

- `<СТАРЫЙ ФОРМАТ> УСТАРЕЛ`
- `<НОВЫЙ СТАНДАРТ> СТАНОВИТСЯ НОРМОЙ`
- `<ПРОФЕССИЯ / РОЛЬ> МЕНЯЕТСЯ`

Examples:

- `КУРС ПО ИИ УМЕР`
- `ВСЕ СООБЩЕСТВА ДОЛЖНЫ БЫТЬ AGENT FIRST`
- `СКИЛЛЫ ВАЖНЕЕ ПОСТОВ`

## Technical terms in headlines

Do not lead with these terms for a broad audience:

- CLI
- MPS
- fp16
- batching
- Flash Attention
- Transformers
- Optimum
- gateway
- orchestration
- MCP, unless the post is specifically about MCP

Instead, move them to the article text or tiny secondary caption only if necessary.

## Final check

Before choosing a headline, ask:

- Would a smart non-technical reader understand this?
- Is the practical value visible without reading the article?
- Is the headline short enough for image generation?
- Could the product name move into a small caption instead of the main line?

If the answer is no, rewrite the headline toward benefit and use case.
