# Reference Gallery

Use this file as a text-based reference library for the best Human 2.0 image directions. It does not replace real visual references, but it gives agents a stable target when the original screenshots are not available.

## How to use

Before generating a cover, inspect the most relevant visual examples in `references/images/`.

If `references/images/` contains a file whose name starts with `TOP Reference`, inspect it first. It is the primary quality bar for premium Human 2.0 Dzen / Telegram covers:

- huge heavy condensed headline with black and brand blue contrast
- expressive editorial object or framed evidence, not a default card grid
- controlled depth, blur, reflections, or material detail when useful
- restrained Human 2.0 palette with brand blue and tiny gold accents
- real H2.0 logo/lockup kept exact and small
- sparse supporting icons/nodes, never a dense process diagram

Use the archetypes below as routing helpers after the TOP Reference, not as permission to ignore the stronger visual benchmark.

Also read `reference-dna.md` for the stronger cross-reference patterns visible in the best covers: crown/object, premium device, evidence collage, real screen hero, and before/after product outcome.

Before generating a cover, choose the closest reference archetype:

1. Editorial workflow cover
2. Product / tool announcement cover
3. Hyperreal premium object cover
4. Clean comparison infographic
5. Reels vertical cover
6. Data strip / compact statistics
7. Notebook / marker concept board

Do not mix more than two archetypes in one image.

## 1. Editorial workflow cover

Best for: posts where the workflow itself is the story. Do not choose this only because the article mentions agents, MCP, skills, automation, or a pipeline.

Look:

- 16:9 light-gray canvas
- huge black headline in a dominant safe-zone block; left placement is common but not required
- one brand-blue keyword in headline
- large white rounded panel in center/right only if it contains real evidence or one clear product surface
- thin brand-blue connector path
- 2-4 clean UI cards or icons max, never as filler
- H2.0 badge bottom-left
- 3x3 dot pattern top-right

Typical composition:

`headline -> input card -> agent/tool node -> result cards`

Avoid: tables, many captions, dense fake UI text, and default `panel + 4 cards` layouts when a hero object or framed screenshot would be more distinctive.

## 2. Product / tool announcement cover

Best for: OpenClaw, Claude Code, Codex, Hermes, HyperFrames, Anthropic, tool updates.

Look:

- product name in headline
- one clean product metaphor
- few cards showing what changed
- white panel with subtle depth
- integrated Human 2.0 mascot as a small generated scene participant when available
- Human 2.0 footer kept small

Good patterns:

- `tool card -> gateway -> output cards`
- `old closed route -> new open routing panel`
- `HTML card -> tool node -> MP4 preview`

Avoid: fake changelog dashboards, huge logos, aggressive versus posters.

## 3. Hyperreal premium object cover

Best for: strong Dzen/Instagram covers where the post needs emotion and clickability.

Look:

- one hero object with shallow depth of field
- white/light-gray background
- controlled blur and shadow
- brand-blue/gold reflections
- minimal floating UI icons
- large headline in the strongest negative-space zone: left, top, right, lower band, or asymmetric overlay

Good objects:

- crown for course/environment shift
- iceberg for hidden ИИ capabilities
- glass robot + human for agent-first community
- physical notebook for orchestrator/workflow map
- glossy control dial for stack/routing/control

Avoid: stock business people, neon cyberpunk, excessive 3D clutter.

## 4. Clean comparison infographic

Best for: before/after, two roles, old vs new standard, data comparison.

Look:

- white/light-gray surface
- 2 columns max
- blue accent on the recommended/new side
- gray or muted old side
- few labels, large numbers
- clear final takeaway
- H2.0 badge bottom-left

Good patterns:

- `без агента` vs `с агентом`
- `agent friendly` vs `agent first`
- `курс` vs `среда`

Avoid: too many rows, tiny captions, generic green/red colors unless data requires them.

## 5. Reels vertical cover

Best for: Instagram Reels / Shorts / vertical social.

Look:

- 9:16 canvas
- headline top, 3-5 short lines
- hero visual center
- brand footer bottom safe zone
- key text inside the central crop
- optional hyperreal object for thumb-stopping effect

Good patterns:

- `ВСЕ СООБЩЕСТВА / ДОЛЖНЫ БЫТЬ / AGENT FIRST`
- human + agent on a glossy platform
- vertical stack of skill/memory/search cards

Avoid: wide diagrams, tiny footers, text at extreme edges.

## 6. Data strip / compact statistics

Best for: article-body visuals with numbers.

Look:

- narrow horizontal or 16:9 layout
- 3-5 white statistic cards
- large brand-blue numbers
- small gray descriptions
- one thin connector or divider line
- source line only if readable

Good patterns:

- information overload stats
- salary premium stats
- adoption gap stats

Avoid: detailed axes when a data card would communicate faster.

## 7. Notebook / marker concept board

Best for: abstract concepts, orchestrator, system maps, frameworks.

Look:

- white notebook/page surface
- subtle grid or paper texture
- blue marker lines
- black marker central label
- clean UI stickers/cards around it
- Human 2.0 badge outside or bottom-left

Good patterns:

- central circle: orchestrator/person
- arrows to agent, memory, skills, sources, people, projects
- sticker-like cards for each node

Avoid: messy handwritten text, too many arrows, low contrast.

## Reference selection rule

When in doubt:

- Article cover: use Premium object cover or Product / tool announcement cover.
- Tool/product news: use Product announcement cover.
- Instagram/Reels: use Reels vertical cover.
- Big conceptual claim: use Hyperreal premium object cover.
- Data/table: use Clean comparison infographic or Data strip.
- Framework/map: use Notebook concept board.

## Quality bar

A good result should feel like a premium editorial slide from the same visual system as the strongest Human 2.0 examples:

- strong headline
- one clear visual metaphor
- lots of air
- white cards
- brand-blue linework
- tiny gold accents
- no generic tech noise
- no tiny fake text
- no broken brand mark
